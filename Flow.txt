III️⃣ Ngoài function — Những thứ quan trọng trước khi làm frontend

Đây mới là phần quan trọng.

1️⃣ Indexing (Performance)

Bạn chắc chắn phải có index cho:

CREATE INDEX idx_exams_user_id ON exams(user_id);
CREATE INDEX idx_exam_questions_exam_id ON exam_questions(exam_id);
CREATE INDEX idx_exam_passages_exam_id ON exam_passages(exam_id);
CREATE INDEX idx_user_answers_exam_id ON user_answers(exam_id);

Nếu không, khi user 1000+:
→ get_exam_detail sẽ chậm.

2️⃣ Unique constraints

Bạn nên có:

UNIQUE (exam_id, exam_question_id) on user_answers

Để tránh 1 question có nhiều answer.

3️⃣ Foreign key cascade rules

Kiểm tra:

exam_questions.exam_id → exams.id ON DELETE CASCADE

exam_passages.exam_id → exams.id ON DELETE CASCADE

exam_question_options.exam_question_id → exam_questions.id ON DELETE CASCADE

Nếu thiếu cascade → orphan data.

4️⃣ Transaction consistency

generate_exam đã ổn.

submit_exam đã lock row.

→ OK.

5️⃣ Data leakage risk

Cần đảm bảo:

review_my_exam không expose is_correct_snapshot cho student trước submit

get_exam_detail không trả is_correct_snapshot

Nếu frontend thấy correct answer trước submit → game over.

6️⃣ Analytics layer (sắp tới)

Sau khi production:
Bạn sẽ cần:

pass rate per question

difficulty auto adjustment

student performance tracking

Nên chuẩn bị schema cho analytics.

IV️⃣ Bạn đã đủ điều kiện làm frontend chưa?

2. Question Engine Architecture

Bạn có 9 engine types.

Production design luôn theo pattern:

Question
   │
   ▼
Engine Type
   │
   ├── options
   ├── text_answer
   ├── reorder
   └── passage
Engine Matrix
Engine	Tables Used
MCQ_SINGLE	questions + options
TEXT_INPUT	questions + question_text_answers
PASSAGE_MCQ	passages + questions + options
PASSAGE_TEXT	passages + questions + question_text_answers
REORDER	questions + question_text_answers
TRUE_FALSE	questions + options
AUDIO_MCQ	questions + options
AUDIO_TEXT	questions + question_text_answers
ESSAY	questions
Engine logic layer
question_types
      │
      ▼
engine_type
      │
      ▼
engine_handler

Ví dụ:

MCQ_SINGLE → mcq_engine
TEXT_INPUT → text_engine
PASSAGE_MCQ → passage_mcq_engine
Trong exam snapshot
exam_questions
      │
      ├── exam_options
      └── exam_passages

Student khi làm bài chỉ đọc:

exam_questions
exam_options
exam_passages

Không đọc question bank.

3. Database Constraint Checklist cho 9 Engines

Đây là checklist production cực quan trọng.

Engine: MCQ

Required:

questions.question_type_id = MCQ

Constraints:

options.question_id NOT NULL
options.option_label UNIQUE per question
>= 2 options
1 correct option
Engine: TEXT_INPUT

Required:

question_text_answers.question_id

Constraints:

accepted_answer NOT NULL
Engine: PASSAGE_MCQ

Required:

questions.passage_id NOT NULL

Flow:

passages
  │
  └── questions
         └── options
Engine: PASSAGE_TEXT

Required:

passage_id NOT NULL
question_text_answers
Engine: REORDER

Required:

question_text
question_text_answers.accepted_answer

Ví dụ:

scramble_text = question_text
correct_text = accepted_answer
Engine: TRUE_FALSE

Options phải có:

TRUE
FALSE

Constraint tốt:

option_label IN ('A','B')
Engine: AUDIO_MCQ

Required:

questions.audio_url
options
Engine: AUDIO_TEXT

Required:

audio_url
text_answer
Engine: ESSAY

Required:

question_text

Optional:

max_score
rubric

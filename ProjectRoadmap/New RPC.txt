# PROJECT MASTER CONTEXT — EXAM PLATFORM

## 1. Project Overview

This project is an **online exam generation and testing platform** built with **Supabase (PostgreSQL)**.

Core capabilities:

* Question bank management
* Passage-based questions (reading comprehension)
* Exam template system
* Automatic exam generation
* Students taking exams
* Answer tracking and scoring
* Teacher statistics dashboard

Architecture direction: **production-level backend-first design**.

---

# 2. Tech Stack

Backend

* Supabase
* PostgreSQL
* Row Level Security (RLS)
* RPC Functions (PostgreSQL functions)

Frontend

# 7. SQL Development Rules

When creating SQL functions:

ALWAYS include a DROP statement before CREATE.

Example:

DROP FUNCTION IF EXISTS function_name;

CREATE OR REPLACE FUNCTION function_name()
RETURNS ...
LANGUAGE plpgsql
AS $$

$$;

Reason:
Avoid duplicate function errors.

---

# 8. RPC Development Workflow

When working with RPC or RLS:

The assistant MUST NOT assume table schemas.

Correct workflow:

1. Identify related tables
2. Request table schemas
3. Then generate SQL logic

---

# 9. RPC Audit Workflow

When auditing RPC functions:

Step 1
List all RPC functions

Step 2
Identify tables used by each RPC

Step 3
Request schema for those tables

Step 4
Audit / refactor / recreate RPC

---

# 10. Code Editing Rules

When modifying code:

The assistant must clearly specify:

* which lines to delete
* exactly where to paste new code
* provide complete code blocks

Never assume the user remembers previous code.

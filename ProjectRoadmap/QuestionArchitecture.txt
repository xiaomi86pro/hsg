Ý nghĩa từng state
State	Ý nghĩa
draft	mới tạo, chưa dùng trong exam
active	có thể dùng trong exam
inactive	tạm khóa, không dùng nhưng giữ data
deleted	soft delete

Function	Transition
create_question	→ draft

deactivate_question	active → inactive

🧱 2️⃣ CÁC LAYER CỐ ĐỊNH (KHÔNG ĐƯỢC THAY ĐỔI)
🔐 A. Security Layer

assert_admin_or_teacher()

(sau này có thể có assert_owner, assert_student…)

Không business logic ở đây.

🧠 B. Type Resolution Layer
get_question_type_id(code)

Single source of truth.

Không được query question_types trực tiếp trong RPC nữa.

📏 C. Validation Layer

Mỗi question type có helper riêng:

validate_mcq_options()
validate_true_false_payload()
validate_open_answer_payload()

RPC chỉ perform validate_xxx(...).

Không validate inline.

💾 D. Data Mutation Layer

Chỉ làm:

insert

update

delete

lock row

optimistic check

Không snapshot ở đây.

📸 E. Snapshot Layer
build_question_snapshot(question_id)

Single source of truth cho audit.

Mọi create/update/delete đều dùng layer này.

📜 F. Audit Layer
insert_question_audit(...)

Không insert trực tiếp vào table nữa.

🔁 3️⃣ QUESTION LIFECYCLE (CHUẨN)
CREATE
assert
→ resolve type
→ validate
→ insert
→ snapshot(new)
→ audit(CREATE_XXX)
UPDATE
assert
→ resolve type
→ lock row
→ optimistic check
→ validate
→ snapshot(old)
→ mutate
→ snapshot(new)
→ audit(UPDATE_XXX)
DELETE (soft delete – sẽ làm sau)
assert
→ resolve type
→ lock row
→ snapshot(old)
→ soft delete
→ audit(DELETE_XXX)
